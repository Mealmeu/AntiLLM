# AntiLLM

리버싱 전용 안티 LLM (C++)

## 개요

이 프로젝트는 리버싱 CTF 문제 풀이 시 사용되는 LLM과 자동화 분석 도구를 낚기 위해 만든 미끼 검증 구조입니다. 단순히 평문을 숨기는 것을 넘어, 분석 환경을 감지해 가짜 플래그로 유도하거나 더미 코드로 디컴파일러의 시간을 낭비하게 만드는 목적을 가집니다.

(주의: 실제 플래그는 소스 코드, README, 리소스, PDB, 실행 파일 내부 등 어디에도 평문으로 포함해선 안 됩니다.)

---

## 플래그 검증 방식

프로그램은 기본적으로 사용자 입력값의 SHA-256 해시를 검증하지만, 해시값을 평범하게 저장하지 않습니다.

1. 정답이 되는 목표 해시를 두 개 이상의 배열(`kDigestPartA`, `kDigestPartB`)로 분할하여 저장합니다.
2. 런타임에 다음과 같이 XOR 연산을 통해 목표 해시를 복원합니다.

```cpp
const std::uint8_t expected = kDigestPartA[index] ^ kDigestPartB[index];

```

3. 입력값의 해시와 복원된 해시가 일치하면 정답 처리됩니다.

검증값을 생성하는 스크립트는 배포 파일에 포함하지 않아야 합니다. 또한 커밋 전 터미널 로그, 빌드 로그, 주석 등에 실제 플래그가 남아있지 않은지 반드시 확인해야 합니다.

---

## 프로그램 실행 흐름

프로그램은 입력을 받은 뒤 다음 순서로 동작합니다.

```text
사용자 입력 > 미끼 문자열 / Maze 코드 참조 > 분석 환경 확인
                                            ㄴ 분석 환경 감지됨 > FakeVerify (가짜 플래그 경로)
                                            ㄴ 일반 환경 > 실제 SHA-256 검증 로직

```

일반 실행 환경에서 입력이 실제 플래그와 일치하면 `Correct`, 틀리면 `Wrong` 메시지가 출력됩니다.

---

## 분석 환경 감지 (Fake Path)

리버서가 디버거를 사용하거나 분석 환경이 감지되면 가짜 검증 경로(`FakeVerify`)가 실행됩니다.

다음 조건 중 하나라도 만족하면 가짜 경로로 빠집니다.

* `IsDebuggerPresent`가 디버거를 감지한 경우
* PEB의 `BeingDebugged` 값이 설정된 경우
* `NtGlobalFlag`의 디버그 관련 값이 설정된 경우
* `CheckRemoteDebuggerPresent`가 디버거를 감지한 경우

분석 환경에서는 가짜 플래그를 검증하는 `FakeVerify()`가 호출됩니다. 리버서나 LLM이 실행 파일의 평문 문자열, 비교 대상, `Correct` 문자열 참조 위치만 단순하게 추적할 경우 무조건 이 가짜 경로에 도달하도록 구성했습니다.

## Maze 코드 동작

`anti_llm_maze.inc`에 정의된 `MazeMix()` 함수는 여러 함수를 간접 호출하며 자동 분석 툴과 LLM의 분석을 방해합니다.

* 의미 없는 연산을 수행해 함수 수와 제어 흐름 복잡도를 증가시킵니다.
* 디컴파일 결과에 노이즈를 추가하여 자동 분석 시간을 크게 늘립니다.
* 실제 검증과 무관한 데이터 흐름을 생성해 실제 검증 로직 추적을 어렵게 만듭니다.

이 코드는 결과적으로 의미 없는 연산이므로 컴파일러 최적화 시 제거될 수 있습니다. 이를 방지하기 위해 `main.cpp`에서는 결과를 `volatile` 변수에 저장합니다.

```cpp
volatile std::uint32_t mazeResult = anlm::MazeMix(seed);

```

## 주요 구현 사항

* 단순 문자열(Strings) 검색 방어
* 자동화된 LLM 분석 교란 및 래빗홀 유도
* 분석 환경 감지를 통한 가짜 풀이 경로 제공
* 더미 연산을 통한 실제 검증 로직 발견 난이도 증가
